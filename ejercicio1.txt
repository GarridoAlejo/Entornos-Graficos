Ejercitación 1 - Responder el siguiente cuestionario
1. Qué es HTML, cuando fue creado, cuáles fueron las distintas versiones y cuál es la última?

Rta: HTML (HyperText Markup Language) es el lenguaje estándar para crear y estructurar páginas web.
No es un lenguaje de programación, sino de marcado, lo que significa 
que usa etiquetas para definir la estructura y el contenido de 
un documento que un navegador puede interpretar 
(por ejemplo, texto, imágenes, enlaces, tablas, formularios).

Fue creado por Tim Berners-Lee en 1991. 
Permitir compartir documentos de investigación entre científicos 
a través de una red llamada World Wide Web (WWW).
La primera versión de HTML se basó en el SGML 
(Standard Generalized Markup Language), un estándar para 
documentos estructurados.
Desde entonces, ha pasado por varias versiones:
1. HTML 1.0 (1991) Muy básico, solo texto, enlaces, imágenes simples.
2. HTML 2.0 (1995) Introdujo formularios y tablas.
3. HTML 3.2 (1997) Añadió soporte para applets, 
scripts y tablas más complejas.
Y demás versiones hasta llegar a:
HTML Living Standard, que es la versión en constante actualización 
y mantenida por el grupo WHATWG (Web Hypertext Application Technology 
Working Group).
Esto significa que HTML siempre está evolucionando 
y no se congela en una versión fija.

2. ¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos
con HTML?

Rta: El W3C (World Wide Web Consortium), que ha sido uno 
de los organismos clave en el desarrollo de HTML, recomienda 
seguir ciertos principios básicos para crear documentos HTML que 
sean correctos, accesibles y fáciles de mantener. Estos principios 
son los siguientes:

- Estructura bien formada y válida

-Separar contenido, presentación y comportamiento

- Uso de etiquetas semánticas

- Accesibilidad para todos los usuarios

- Uso correcto de enlaces e hipervínculos

- Compatibilidad y portabilidad

- Optimización y peso ligero

- Metadatos y codificación


3. En las Especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿y
obsoleto?

Rta: En las Especificaciones de HTML 
(como las publicadas por el W3C y el WHATWG), 
los términos desaprobado y obsoleto tienen significados diferentes, 
aunque a veces la gente los confunde.

Se considera desaprobado cuando sigue estando definido en 
la especificación para mantener compatibilidad con documentos antiguos, 
pero ya no se recomienda su uso en nuevos documentos.
Todavía existe, pero no deberías usarlo (puede romperse en el futuro).

Se considera obsoleto cuando ya no forma parte del estándar vigente 
y su uso no es válido en HTML moderno.
Ya no existe en el estándar; usarlo es incorrecto y puede no funcionar.

4. ¿Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?

Rta: El DTD (Document Type Definition, o Definición de 
Tipo de Documento) es una parte fundamental de las versiones antiguas 
de HTML, especialmente hasta HTML 4.01 y XHTML.
Es un conjunto de reglas que define la estructura y 
los elementos permitidos en un documento HTML.Indica al navegador 
qué versión de HTML se está usando y qué gramática debe seguir 
para interpretar el documento. Se declara en la primera línea 
del archivo HTML mediante <!DOCTYPE ...>.

En HTML moderno (HTML5 y el estándar vivo) ya no se usan DTDs 
completos, sino una declaración simplificada:

<!DOCTYPE html>

pero en HTML 4.01 eran obligatorios y más complejos.

Para resumir , los DTDs de HTML 4.01 son:

Strict → Código limpio, sin etiquetas de presentación.

Transitional → Más permisivo, mantiene compatibilidad con etiquetas antiguas.

Frameset → Igual que Transitional, pero para páginas con marcos.

5. ¿Qué son los metadatos y cómo se especifican en HTML?

Rta: Los metadatos son información sobre el documento web que 
no forma parte del contenido visible de la página, 
pero que describe sus características, configuración o comportamiento.
Sirven para que navegadores, motores de búsqueda, redes sociales 
y otros sistemas interpreten correctamente la página.

Se pueden especificar con la etiqueta `<meta>` dentro del `<head>` 
del documento HTML; con la etiqueta <link> para enlaces a 
recursos externos; con la etiqueta `<title>` para el título 
de la página; O mediante atributos en <html> o <body>.
